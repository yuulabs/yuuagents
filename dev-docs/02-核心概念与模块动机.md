# yuuagents：核心概念与模块动机（why-first）

## 目标与约束

- 目标：把 “Persona + Tools + LLM” 变成可部署、可观察、可脚本化的 Linux 服务。
- 约束：CLI 优先；复用现有的 yuu 生态（yuutools/yuullm/yuutrace）；默认在 Docker 中执行有副作用的操作。

## 核心概念

### 1) Task（一次运行）与 task_id

- why：daemon 需要同时管理多个并行任务，并且要能查询状态/历史/成本。
- 做法：每次提交任务生成 `task_id`（uuid4 hex），API 与 CLI 的主索引键都是 `task_id`。
- 关联实现
  - 任务请求 DTO：[types.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/types.py)
  - 任务生命周期管理：[manager.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/manager.py)
  - REST API 路由：[api.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/api.py)

### 2) Agent（一次任务内的对话实体）与 agent_id

- why：同一种 persona/tools/model 组合可以被复用为一个 “agent 配置名”，而每次运行是一个 “task”。
- 做法：`agent_id` 实际上是配置里的 agent 名（例如 `main`/`coder`），而不是全局唯一实例 ID。
- 关联实现
  - Agent 领域模型（config/state 分离）：[agent.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/agent.py)
  - 任务提交时将 `req.agent` 写入 `AgentConfig.agent_id`：[manager.py:L52-L104](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/manager.py#L52-L109)

### 3) Persona（system prompt 的主体）

- why：把 “角色设定/工作方式/约束” 固化到系统提示，减少每个任务重复描述。
- 做法：persona 可以来自配置 `agents.<name>.persona`，也可以被 `yagents run --persona` 覆盖。
- 关联实现
  - system prompt 组装器：`SimplePromptBuilder`：[agent.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/agent.py)
  - persona 分辨优先级（显式 override > agent config > 回退到 agent 名）：[manager.py:L67-L72](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/manager.py#L67-L72)

### 4) Tools（可调用能力）与 AgentContext（依赖注入）

- why：Agent 需要可控的副作用入口（读写文件、执行命令、网络请求），并且工具需要运行时上下文（容器 ID、workdir、api key）。
- 做法：
  - 工具用 `yuutools @tool` 定义，参数稳定、语义明确。
  - 工具通过 `yt.depends(lambda ctx: ...)` 从 `AgentContext` 注入依赖。
- 关联实现
  - 上下文对象：[context.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/context.py)
  - 内置工具注册表：[tools/__init__.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/tools/__init__.py)
  - 工具示例：`execute_bash`、`read_file/write_file/delete_file`、`web_search`
    - [bash.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/tools/bash.py)
    - [file.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/tools/file.py)
    - [web.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/tools/web.py)

### 5) Docker 隔离（工具执行环境）

- why：把高风险操作（bash、文件修改）放进容器，降低对宿主机的破坏面；同时保持可读的宿主机视图（只读挂载）。
- 做法：
  - 宿主机根目录只读挂载到 `/mnt/host`。
  - 每个容器有自己的持久写目录映射到容器内 `/home/yuu`。
  - 默认共享一个长期存活的容器（`yagents-default`），也支持为单个任务创建临时容器（`--image`）。
- 关联实现
  - Docker manager：[docker.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/docker.py)
  - 注入到 system prompt 的容器说明：`DOCKER_SYSTEM_PROMPT`：[docker.py:L17-L24](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/docker.py#L17-L24)

### 6) Flow/Agent 执行模型（LLM → 工具 → LLM）

- why：把 agent 行为约束成可追踪的最小闭环，便于插桩、成本统计、失败定位。
- 做法：
  - Flow 是最小执行单元（可观测、可寻址、可中断），Agent 在 Flow 之上组合 LLM 行为。
  - 使用 `yuullm` 的 stream API 得到文本片段与 tool calls。
  - tool calls 解析为并发执行的调用列表，把结果追加回 history。
  - 最终没有 tool calls 时视为 DONE。
- 关联实现
  - Flow + Agent：[core/flow.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/core/flow.py)
  - Session（Flow 的薄包装）：[runtime_session.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/runtime_session.py)

### 7) Daemon + CLI（可脚本化的控制面）

- why：agent 是长任务；需要可查询、可取消、可收集历史的控制面；CLI 需要稳定可集成。
- 做法：
  - daemon：Starlette + uvicorn，监听 Unix Domain Socket。
  - CLI：httpx uds transport 访问 `http://yagents/...` 路由。
- 关联实现
  - CLI 入口与命令：[cli/main.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/cli/main.py)
  - CLI thin client：[cli/client.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/cli/client.py)
  - daemon server：[server.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/server.py)
  - REST API：[api.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/api.py)

### 8) Config（运行时可变的控制参数）

- why：provider/model/persona/tools 需要可配置，且在 daemon 运行中可调整。
- 做法：YAML + msgspec convert；写盘后调用 `/api/config/reload` 做热重载。
- 关联实现
  - 配置结构与加载：[config.py](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/config.py)
  - config reload API：[api.py:L122-L145](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/api.py#L122-L145)
  - manager 热重载入口：[manager.py:L156-L167](file:///home/tomorrowdawn/project/yuulabs/agent-kits/yuuagents/src/yuuagents/daemon/manager.py#L156-L167)

## 模块与概念映射（快速索引）

| 概念 | 主要模块 | 为什么在这里 |
|---|---|---|
| Agent config/state | `src/yuuagents/agent.py` | 把不可变配置与运行时状态分离，便于管理与观测 |
