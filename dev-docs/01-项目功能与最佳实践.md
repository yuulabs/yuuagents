# yuuagents：项目功能与最佳实践

## 功能定位

`yuuagents` 当前可以分成两条路径：

- SDK 路径：`import yuuagents`，在本进程里直接跑 agent。
- daemon 路径：`yagents install` 初始化本地状态，再用 `yagents up` 启动 Unix Socket 上的 HTTP daemon。

代码层面，SDK 和 daemon 共享同一套 `Session`、`AgentState`、工具注册和 capability 约束，只是运行位置不同。

## 当前真实入口

| 命令 | 当前行为 |
|---|---|
| `yagents install` | 写入 `~/.yagents/config.yaml`，创建目录，初始化 SQLite，准备 Docker 镜像 |
| `yagents up` | 启动 daemon。前台模式会启动 yuutrace server 和 UDS HTTP server |
| `yagents up -d` | 优先尝试 systemd user service；否则后台拉起子进程 |
| `yagents run` | 提交一个任务到 daemon |
| `yagents list` | 打印任务摘要 |
| `yagents status <task_id>` | 输出单个任务的 JSON 状态 |
| `yagents logs <task_id>` | 输出会话历史 |
| `yagents input <task_id> "..."` | 向运行中的任务注入一条用户消息 |
| `yagents stop <task_id>` | 取消任务 |
| `yagents config` | 打印当前磁盘配置；可更新配置并尝试热重载 |
| `yagents trace ui` | 打开 yuutrace Web UI |
| `yagents down` | 请求 daemon 退出，并尝试停止 systemd user service |
| `yagents uninstall` | 删除 `~/.yagents` 及相关持久化状态 |

全局参数里，`yagents --socket` 可以覆盖 daemon Unix socket 路径。

## 安装前提

- Python 3.14+
- 基础包已经包含 CLI、daemon bootstrap、YAML、SQLite 和 UDS HTTP 所需依赖
- 如果要使用 Docker 工具，必须额外安装 `yuuagents[docker]`，并确保 Docker Engine 可达
- 如果要使用 `web_search`，必须额外安装 `yuuagents[web]`，并提供 Tavily API key
- daemon 前台启动时依赖 `ytrace` 可执行文件

## `yagents install`

`install` 的合并顺序和旧文档不一样，当前实现是：

1. 从包内 bundled default template 开始
2. 合并当前工作目录里的 `config.overrides.yaml`
3. 合并 `--project-dir` 指向目录里的 `config.overrides.yaml`
4. 合并 `--overrides`
5. 如果传了 `--config`，则直接改为读取那个完整配置，并要求确认

补充行为：

- 默认安装路径会把 `docker.image` 改成当前包版本对应的 runtime 镜像 tag
- `--config` 分支不会再套用当前目录或 `--project-dir` 的 overrides
- 配置写入目标始终是 `~/.yagents/config.yaml`
- 之后会创建 `~/.yagents/dockers/` 和 SQLite 父目录
- 只有当配置里实际启用了 Docker 工具时，才会检查镜像构建/拉取
- Docker 不可用时，install 会发警告并继续，不会强制退出

如果传了 `--systemd`，install 还会调用 `~/.config/systemd/user/yagents.service` 的注册逻辑，实际执行的是 `systemctl --user`。

## `yagents up`

`up` 的真实配置解析顺序如下：

1. 若传 `--config`，直接加载该文件
2. 否则优先找当前目录的 `config.yaml`
3. 若没有，再用已经安装的 `~/.yagents/config.yaml`
4. 如果这些都没有，则回退到包内 bundled default template
5. 当前目录的 `config.overrides.yaml` 会继续叠加

另外：

- `--dot-env` 会显式加载指定 `.env`
- 不传 `--dot-env` 时，会从当前目录向上查找 `.env`，一直走到 `$HOME`
- 若新配置的数据库路径与已安装配置不同，`up` 会拒绝启动，避免把旧 SQLite 文件留在磁盘上
- `-d/--daemon` 在没有 `.env` 的情况下优先走 systemd user service，否则回退到子进程后台启动
- 前台 `up` 会先拉起 yuutrace server，再起 UDS HTTP server

注意：`up` 目前没有像 `install` 那样做完整的配置验证阻断，所以手工改坏的配置可能先通过加载，直到运行时才报错。

## `yagents run`

`run` 通过 HTTP API 提交任务，当前 payload 包括：

- `agent`
- `persona`
- `input`
- `model`
- `container`
- `image`
- `tools`

约束点：

- `--task` 必填
- `--tools` 是逗号分隔的工具名列表
- `--container` 和 `--image` 不能同时给，Docker resolver 会直接报错
- 未知工具名不会被静默忽略，最终会在运行时失败

## `yagents config`

无参数时：

- 若 `~/.yagents/config.yaml` 存在，就读取它并打印摘要
- 若不存在，就回退到 bundled default template

带参数时：

- `--config` 会完全替换当前配置
- `--overrides` 会在当前配置上做 deep merge
- 写盘后会尝试调用 daemon 的 `/api/config/reload`
- 如果 daemon 没有运行，只会提示并保留磁盘变更

## `yagents list/status/logs/input/stop`

- `list` 输出人类可读摘要，不是 JSON
- `status` 输出 JSON
- `logs` 按 role 分段打印历史消息
- `input` 发送一条用户消息到运行中的任务
- `stop` 取消任务并让 daemon 更新状态

## 工具能力

当前包内工具名如下：

- `sleep`
- `view_image`
- `delegate`
- `inspect_background`
- `cancel_background`
- `input_background`
- `defer_background`
- `wait_background`
- `execute_bash`
- `read_file`
- `edit_file`
- `delete_file`
- `web_search`

能力边界：

- `sleep`、`view_image` 不依赖 daemon
- `delegate` 和后台控制工具需要 daemon 注入 delegate capability
- Docker 工具需要 Docker capability 和 `yuuagents[docker]`
- `web_search` 需要 web capability、`yuuagents[web]` 和 Tavily key

## 配置约定

当前代码中的结构化配置由 `msgspec.Struct` 定义，核心默认值在 `src/yuuagents/config.py`，包内默认模板在 `src/yuuagents/config.default.yaml`。

重要默认值：

- `~/.yagents/config.yaml`
- `~/.yagents/tasks.sqlite3`
- `~/.yagents/traces.db`
- `~/.yagents/yagents.sock`

`Config.validate()` 目前主要检查：

- `db.url` 不能为空
- `yuutrace.db_path` 不能为空
- `yuutrace.ui_port`、`yuutrace.server_port` 必须在 1..65535
- `snapshot.restore_on_start` 需要 `snapshot.enabled = true`
- agent 的 `description` 不能为空
- agent 引用的 provider 和 subagent 必须存在

不过要注意，`install` 和 `config` 在当前实现里更偏向提示 warning，不是所有配置错误都会在命令层面硬失败。

## 运行建议

- 配置路径和运行路径不要混用，安装态写进 `~/.yagents/config.yaml`，源码仓库里的 reference template 只是模板
- 变更 `db.url` 时先想清楚数据迁移策略，因为 `up` 会阻止直接切换
- 如果 agent 配了 Docker 工具，要同时确认 Docker 可达和镜像可用
- 如果 agent 配了 `web_search`，要同时确认 Tavily key 和 `web` extra
